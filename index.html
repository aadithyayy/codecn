üß© 1. TCP Program

Algorithm ‚Äì TCP Server

  1. Start the program.
  2. Create a socket using socket(AF_INET, SOCK_STREAM, 0).
  3. Assign server IP and port number to sockaddr_in.
  4. Bind the socket using bind().
  5. Listen for connections using listen().
  6. Accept connection using accept().
  7. Receive message using recv().
  8. Send response using send().
  9. Close the connection using close().

Algorithm ‚Äì TCP Client

  1. Start the program.
  2. Create socket using socket(AF_INET, SOCK_STREAM, 0).
  3. Assign server IP and port using sockaddr_in.
  4. Connect using connect().
  5. Send message using send().
  6. Receive response using recv().
  7. Close socket using close().
üß© 2. UDP Program

Algorithm ‚Äì UDP Server

  1. Create socket using socket(AF_INET, SOCK_DGRAM, 0).
  2. Bind using bind().
  3. Receive message with recvfrom().
  4. Send response with sendto().
  5. Close socket.

Algorithm ‚Äì UDP Client

  1. Create socket using socket(AF_INET, SOCK_DGRAM, 0).
  2. Send message using sendto().
  3. Receive response using recvfrom().
  4. Close socket.
üß© 3. Stop-and-Wait ARQ (UDP)

Reliable data transfer over UDP ‚Äî send one frame, wait for ACK before sending the next.

Algorithm ‚Äì Sender

  1. Create UDP socket.
  2. Initialize sequence number = 0.
  3. Send frame using sendto().
  4. Wait for ACK with timeout.
  5. Retransmit if timeout or wrong ACK.

Algorithm ‚Äì Receiver

  1. Create UDP socket and bind it.
  2. Receive frame using recvfrom().
  3. Send ACK for correct frame.
  4. Resend last ACK for duplicate frames.
üß© 4. Go-Back-N ARQ (UDP)

Sender sends multiple frames before waiting for ACK; receiver only accepts in-order frames.

Algorithm ‚Äì Sender

  1. Set window size N.
  2. Send N frames without waiting.
  3. Wait for ACKs and slide window.
  4. Retransmit on timeout.

Algorithm ‚Äì Receiver

  1. Receive frame and verify sequence.
  2. Send ACK for correct frame.
  3. Discard out-of-order frames.
üß© 5. Selective Repeat ARQ (UDP)

Multiple frames sent; only erroneous frames are retransmitted.

Algorithm ‚Äì Sender

  1. Set window size N.
  2. Send frames and start timers.
  3. Slide window when ACK received.
  4. Retransmit only lost frames.

Algorithm ‚Äì Receiver

  1. Receive and buffer in-window frames.
  2. Send ACK for each frame.
  3. Deliver frames in order.
