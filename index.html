üß© 1. TCP Program

Algorithm ‚Äì TCP Server

  1. Start the program.
  2. Create a socket using socket(AF_INET, SOCK_STREAM, 0).
  3. Assign server IP and port number to a structure (struct sockaddr_in).
  4. Bind the socket using bind().
  5. Listen for incoming connections using listen().
  6. Accept connection from client using accept().
  7. Receive message from client using recv().
  8. Process or display the message.
  9. Send response to client using send().
  10. Close the connection using close().
  11. Stop.

Algorithm ‚Äì TCP Client

  1. Start the program.
  2. Create socket using socket(AF_INET, SOCK_STREAM, 0).
  3. Assign server IP and port number to structure (struct sockaddr_in).
  4. Establish connection to server using connect().
  5. Send message to server using send().
  6. Receive response from server using recv().
  7. Display the message.
  8. Close socket using close().
  9. Stop.

üß© 3. Stop-and-Wait ARQ (UDP)

Reliable data transfer over UDP ‚Äî send one frame, wait for ACK before sending the next.

Algorithm ‚Äì Sender

  1. Start.
  2. Create UDP socket.
  3. Initialize sequence number = 0.
  4. Read data frame from user.
  5. Send frame to receiver using sendto().
  6. Wait for ACK with a timeout:
    • If ACK received and correct ‚Üí toggle sequence number and send next frame.
    • If timeout or wrong ACK ‚Üí retransmit the same frame.
  7. Repeat until all frames are sent.
  8. Close socket.
  9. Stop.

Algorithm ‚Äì Receiver

  1. Start.
  2. Create UDP socket and bind it.
  3. Wait for data frame using recvfrom().
  4. If received frame has expected sequence number:
    • Display data.
    • Send ACK for that sequence number.
    • Toggle expected sequence number.
  5. Else (duplicate frame) ‚Üí resend previous ACK.
  6. Repeat until all frames are received.
  7. Close socket.
  8. Stop.

üß© 4. Go-Back-N ARQ (UDP)

Sender sends multiple frames before waiting for ACK; receiver only accepts in-order frames.

Algorithm ‚Äì Sender

  1. Start.
  2. Create UDP socket.
  3. Set window size N.
  4. Read and prepare data frames.
  5. Send up to N frames sequentially (without waiting).
  6. Wait for ACKs:
    • If ACK for oldest frame received ‚Üí slide window and send next frame.
    • If timeout occurs ‚Üí retransmit all frames in current window.
  7. Repeat until all frames are acknowledged.
  8. Close socket.
  9. Stop.

Algorithm ‚Äì Receiver

  1. Start.
  2. Create and bind UDP socket.
  3. Expect sequence number = 0.
  4. Receive frame:
    • If correct ‚Üí display data, send ACK for next expected frame.
    • If incorrect (out of order) ‚Üí discard frame and resend last ACK.
  5. Repeat until all frames are received.
  6. Close socket.
  7. Stop.

üß© 5. Selective Repeat ARQ (UDP)

Multiple frames sent; only erroneous frames are retransmitted.

Algorithm ‚Äì Sender

  1. Start.
  2. Create UDP socket.
  3. Set window size N.
  4. Send frames within window.
  5. Start individual timer for each frame.
  6. For each ACK received:
    • Mark frame as received and slide window forward.
  7. If timeout occurs for any frame ‚Üí retransmit only that frame.
  8. Repeat until all frames are acknowledged.
  9. Close socket.
  10. Stop.

Algorithm ‚Äì Receiver

  1. Start.
  2. Create and bind UDP socket.
  3. Maintain window of expected sequence numbers.
  4. Receive frame:
    • If within window ‚Üí store and send ACK for that frame.
    • If duplicate ‚Üí resend ACK.
  5. Deliver frames to application in order.
  6. Slide window forward as consecutive frames are delivered.
  7. Repeat until all frames are received.
  8. Close socket.
  9. Stop.

üîë Key Points

TCP

  • Connection-oriented and reliable.
  • Uses 3-way handshake.
  • Ensures ordered and error-free delivery.
  • Server: listen() and accept().
  • Client: connect().

UDP

  • Connectionless, faster but unreliable.
  • No handshake or guaranteed delivery.
  • Independent datagrams, best effort.

Stop-and-Wait

  • One frame at a time; wait for ACK.
  • Retransmit on timeout.
  • Sequence numbers alternate (0 & 1).

Go-Back-N

  • Sends multiple frames (window size N).
  • Retransmits all unacknowledged frames on timeout.
  • Receiver only accepts in-order frames.

Selective Repeat

  • Both sides maintain sliding windows.
  • Only erroneous frames retransmitted.
  • Receiver buffers out-of-order frames.
